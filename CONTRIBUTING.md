# Contributing

Thank you for helping improve Claude Code Operating System.

## What this project accepts

- Clearer documentation
- Better examples
- Safer hook patterns
- Useful starter templates
- Corrections when Claude Code behavior changes
- Practical examples from real long-running projects

## Contribution guidelines

- Keep guidance practical and repository-oriented.
- Prefer small, focused pull requests.
- Avoid turning `CLAUDE.md` examples into large manuals.
- Do not include secrets, private prompts, customer data, or production credentials.
- When adding a workflow, explain when it should and should not be used.
- If a feature depends on current Claude Code behavior, mention that it may need periodic review.

## Documentation style

- Use plain language.
- Prefer tables and short checklists for operational guidance.
- Keep English and Turkish documentation conceptually aligned.
- Use examples that can be copied into real projects.

## Local review checklist

Before opening a pull request:

1. Review changed Markdown files for broken links.
2. Confirm examples do not reference private systems.
3. Confirm shell snippets avoid destructive commands.
4. Update both English and Turkish docs when the main concept changes.

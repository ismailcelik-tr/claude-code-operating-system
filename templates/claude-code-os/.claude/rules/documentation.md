# Documentation Rules

- Update user-facing docs when behavior changes.
- Update `PROJECT_CONTEXT.md` when architecture or product context changes permanently.
- Update `SESSION_CONTEXT.md` at the end of meaningful sessions.
- Update `docs/IMPLEMENTATION_STATE.md` when project progress changes.
- Update `docs/OPEN_DECISIONS.md` when unresolved decisions appear or are resolved.
- Keep `CLAUDE.md` concise; do not store temporary task notes there.
